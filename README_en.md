## üåü Overview
**Exho** is a multimodal intelligent LLM-based Discord bot that supports both text and image inputs.  
It can chat naturally, analyze images, answer questions, assist with coding, and even generate creative content.  
It runs entirely on cloud-based LLMs (like Cohere AI), meaning **no local models are needed** ‚Äî bringing real-time AI experiences directly to Discord.

---

## üöÄ Main Features
- **Smart Conversations**: Understands and responds naturally, capable of roleplay or tone adaptation.  
- **Image Understanding**: Analyzes images, recognizes text, and generates descriptions.  
- **Coding Assistant**: Writes code samples, helps debug, and gives coding advice.  
- **Multi-Model Switching**: Automatically selects the best LLM model for different tasks.  
- **Interactive Games**: Simple text-based games, adventures, and Q&A challenges.  
- **Multimodal Integration**: Supports text + image + (future voice) input/output.

## ‚öôÔ∏è Configuration

Follow these steps to set up your bot.

### Step 1: Configure the `.env` file

Rename `.env.Example` to `.env` and fill in your details:

```env
# .env

DISCORD_TOKEN=Your-Discord-Bot-Token
COHERE_API_KEY=Your-Cohere-API-Key
URL_CHECK_API_KEY=Optional
IP_LOOKUP_API_KEY=Optional
GEMINI_API_KEY=Your-Gemini-API-Key

ADMIN_ROLE_IDS=Admin-Role-IDs

MEMORY_LIMIT=AI-context-limit, e.g. 50

TEXT_MODEL=command-r-03-2025
VISION_MODEL=gemini-2.0-flash

BOT_VERSION=V1
SERVER_LINK=Your-support-server-link

DEV_USER=Developer1-ID,Developer2-ID (and so on)

BAN_CHANNEL=Channel-ID-for-ban-notifications

PREFIX_REPLY=Enable-or-disable-prefix-commands
PREFIX=Prefix-symbol, e.g. "$" so the command looks like $status

PTERO_API_KEY=Your-Pterodactyl-Panel-User-API-Key
PTERO_URL=Your-Pterodactyl-Panel-URL (include https:// or http://)
SERVER_ID=Pterodactyl-Server-ID

REDIS_HOST=Redis-Server-IP
REDIS_PORT=Redis-Server-Port
REDIS_PASS=Redis-Password

DB_USER=MariaDB-Username
DB_NAME=MariaDB-Database-Name
DB_HOST=MariaDB-Server-IP
DB_PORT=MariaDB-Server-Port
DB_POOL_MAX=Max-Connection-Pool, e.g. 20
```

---

### Step 2: Check your `./data` folder

Make sure it contains the following files:

1. `banlist.json` = `[]`  
2. `report_time.json` = `{}`  
3. `last_status_message.json` = `{}`  
4. `marriage.json` = `{}`  
5. `used_command.txt` = leave empty (auto-generated by the bot, but make sure it exists)

---

### Step 3: Run the Bot

Once setup is complete, start the bot with:

```bash
node bot.js
```

üìù **Note:**  
You can update the bot anytime using:

```bash
node update.js
```
